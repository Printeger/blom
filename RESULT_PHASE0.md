## Step 2. 计算全局 MINCO 标量参考解

### 运行方式

通过 Python 直接调用：

```python
setup = BLOMProblemSetup.make_random(M=6, seed=42, q_scale=0.8)
minco = solve_minco_scalar(setup)
```

### 功能说明

这一步求解 Phase 0 的全局 minimum-snap 参考问题。  
它输出：

- 每段 7 次多项式系数
- 全局总代价
- 后续可被 `trajectory_eval.py` 和 `phase0_checks.py` 继续消费的结果对象

- `cost = 151.01570657608463`

```text
 -6.150042635103468e-19, -0.0006275697086888028]
```

- 第 3 段系数：

```text
[0.012147335456314523, 1.3194349323016703, 0.2086713507193045,
 -0.7352716630128164, -0.19527036883337695, 0.09106970975646948,
```

### 分析

- `coeffs.shape = (6, 8)` 说明每段都被表示成 8 个系数的 7 次多项式，和 `2s - 1 = 7` 完全一致。
- 总代价为正，说明该解不是退化零解，同时满足 minimum-snap 问题应有的非负性。
- 系数中高阶项并不全为零，说明全局解确实在利用 7 次多项式的自由度来满足多段插值与高阶连续性约束。

## Step 4. 计算局部 BLOM 窗口解

### 运行方式

```python
local = solve_blom_local_qp(setup, center_segment=3)
```

### 功能说明

这一步求解 `center_segment = 3` 的局部窗口问题。  
它只对窗口 `W(3, 2)` 内的段做局部求解，并返回：

- 活动窗口段索引
- 窗口内全部系数
- 中心段系数
- 局部代价

### 结果

- `center_segment = 3`
- `active_segments = [2, 3, 4]`
- `window_coeffs.shape = (3, 8)`
- `cost = 0.0`
- 中心段系数：

```text
[0.012147335456314523, 1.21101707906002, 0.0690740034580577,
 -0.5368835649713634, 0.0, 0.0, 0.0, 0.0]
```

### 分析

- 窗口确实是 `3-point stencil`，索引 `[2, 3, 4]` 正好对应中心段两侧的局部邻域。
- 局部代价为 `0.0`，这不是错误，而是当前 canonical local 问题的一个可解释结果：  
  该局部窗口在给定插值、连续性和自然边界条件下，存在一个高阶项全为零的分段三次解，因此 `p''''(t) = 0`，积分代价自然为 0。
- 中心段系数的 4 到 7 次项都为 0，也直接支持了上面的解释：该局部最优解退化成 cubic，而 cubic 的 snap 恒为 0。
- 这说明当前 `blom_local_qp.py` 在数值上是自洽的，但也提示我们：后续 Phase 1/2 如果要研究更强的 BLOM 行为，需要引入更丰富的局部结构或更强的比较框架。
