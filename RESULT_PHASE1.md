# RESULT_PHASE1

本文档记录对 `phase_1/` 目录下 Phase 1 代码的实际运行结果。  
与 `RESULT_PHASE0.md` 一样，本文档按“步骤 -> 功能说明 -> 实际结果 -> 分析”的结构整理。

本版结果已在 Phase 2 完成后重新执行并复核，确保 Phase 1 baseline 作为
Phase 2 必要性验证后端时仍然可复现、可运行、可验证。

Phase 1 和 Phase 0 的关键区别是：

- Phase 0 的全局参考实现主要服务于骨架验证；
- Phase 1 的 `minco_scalar_baseline.py` 实现的是一个更严格的全局 MINCO 母问题；
- 除了 `q` 和 `T` 之外，Phase 1 还固定了初末端 boundary jet：
  `zeta_start = (p(0), p'(0), p''(0), p'''(0))`
  `zeta_end = (p(T_total), p'(T_total), p''(T_total), p'''(T_total))`

本次运行中，主验证案例使用：

- `q = [0.0, 0.2437736638035451, -0.5882136211888513, 0.012147335456314523, 0.7645991085692857, -0.7962290423537836]`

- `zeta_end = [-0.7962290423537836, 0.0, 0.0, 0.0]`

也就是一个 `M = 5` 的标量 minimum-snap baseline 问题。

## Step 2. 构造 Phase 1 母问题线性系统

### 运行方式

通过 Python 调用：

```python
M_mat = build_system_matrix(T)
b_vec = build_rhs(q, zeta_start, zeta_end)
```

### 功能说明

这一步把 Phase 1 的数学对象写成线性系统：

```text
M(T) c = b(q, zeta_start, zeta_end)
```

其中：

- `M(T)` 只依赖时间向量 `T`
- `b(...)` 包含 waypoint 和边界 jet 数据
- `c` 是所有段的 7 次多项式系数堆叠向量

### 结果

- `M_seg = 5`
- `T_total = 4.9965016020710005`
- `M_mat.shape = (40, 40)`
- `b_vec.shape = (40,)`
- `rank(M_mat) = 40`
- `cond(M_mat) = 107403.08584332098`
- `M_mat` 与 `b_vec` 中所有元素均为有限值

### 分析

- 每段有 8 个系数，因此总未知量是 `8 * 5 = 40`，系统矩阵是 `40 x 40`，尺寸正确。
- `rank(M_mat) = 40` 说明在线性代数意义上该系统是满秩的，这正是 Phase 1 文档里“固定 boundary jet 以保证唯一性”的数值体现。
- 条件数约为 `1.07e5`，对一个高阶多项式系统来说不算离谱，说明系统虽然不是极度理想，但仍处于可稳定求解范围。
